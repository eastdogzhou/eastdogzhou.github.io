阶段一：预训练（Pre-training）

知识的获取 - 规模令人震撼的工程

训练规模

数据量：约10TB文本
相当于1000万本书

计算资源：6000个GPU × 12天

成本：约200万美元
最先进模型可能需要上亿美元

参数量：700亿个参数
(Llama 2 70B)

预训练的本质

互联网文本(10TB)

↓ 神经网络压缩

模型参数(140GB)

压缩比 ~100:1

这个过程中，模型学会了：

  • 知识提取：记住重要的事实和概念
  • 模式识别：理解语言的深层规律
  • 推理能力：学会简单的逻辑推导

阶段二：监督微调（SFT）

从生成器到助手

预训练后的模型像是一个"博学但不会对话的学者"
SFT让它学会如何有效交流

微调数据的特点

  • 数量少但质量高：
    约10万个高质量对话（vs 10TB预训练数据）
  • 人工标注：
    雇佣专业标注员创建理想的问答对
  • 涵盖多种任务：
    问答、写作、编程、分析等

强化学习优化

让模型生成更高分的回答，持续改进

Scaling Law：大力出奇迹的科学依据

2020年，OpenAI发现了LLM的"物理定律"

Loss = f(N, D)

N = 模型参数量

D = 训练数据量

关键洞察

  1. 平滑提升：性能随规模平滑增长，没有突然的跃变
  2. 无饱和迹象：目前还没看到性能上限
  3. 最优比例：参数量和数据量需要平衡增长

Scaling Law的实际影响

模型规模 能力表现
1B参数 基础语言理解，简单问答
10B参数 复杂推理，多轮对话
100B参数 专业知识，创造性任务
1T+参数 接近人类专家水平

对生信研究的启示

  • 更大的模型 = 更强的生物学知识理解
  • 专门的生物医学大模型正在开发中
  • 未来可能出现"AI生物学家"

通用数据大模型（LDM）：数据领域的GPT

当Scaling Law遇上数据分析

什么是LDM？

类似LLM，但专注于数值数据而非文本
基于Transformer架构，在海量真实数据集及结构因果合成数据集上训练
具备强大的数据理解、预测及泛化能力

LLM: Next Token Prediction

LDM: Missing Value Prediction

核心突破

  • 解决LLM数值短板：如9.11 vs 9.9比较
  • 学习因果结构：而非简单关联
  • 通用性验证：跨领域数据均有效

LDM技术架构

LDM-V2 6000万 超过SOTA
规划中 10亿→100亿 显著超出

当前的局限性

幻觉现象

可能生成看似合理但实际错误的信息

  • 虚构的基因名称或不存在的研究结果
  • 原因：模型是在"插值"而非"记忆"

知识时效性

训练数据有截止日期

  • GPT-4的知识截止于2023年
  • 无法获知最新的研究进展

计算精度

不擅长精确的数值计算

  • 需要调用外部工具进行统计分析
  • 这也是为什么需要AI+BI结合

深度推理

缺乏"System 2"思维

  • System 1：快速、直觉（当前LLM）
  • System 2：慢速、深思（未来方向）

从LLM到智能Agent

让AI从"回答问题"到"解决问题"

  • 知识截止于训练时间

  • 缺乏工具使用能力

Agent的突破

  • 实时检索外部知识

Deep Research：从查询到研究

传统搜索 vs Deep Research

传统搜索 Deep Research
一次查询 迭代式探索
返回链接列表 生成综合报告
需要人工整合 自动整合分析

Deep Research的核心流程

  1. 理解问题：不只是关键词，而是理解研究意图
  2. 制定计划：分解为多个子问题
  3. 迭代查询：根据发现调整搜索策略
  4. 综合分析：整合多源信息，生成洞察

多组学整合Agent

多组学数据的复杂性

组学层次：
├── 基因组 (DNA变异)
├── 表观组 (甲基化、染色质)
├── 转录组 (RNA表达)
├── 蛋白组 (蛋白丰度)
└── 代谢组 (代谢物)

每层都有不同的：
- 数据格式
- 数值范围
- 生物学意义

工作示例

用户："为什么这个基因RNA高表达但蛋白很少？"

Agent分析过程：

1. 检查RNA数据 ✓ 确认高表达

2. 检查蛋白数据 ✓ 确认低丰度

3. 查看甲基化 → "启动子无甲基化，排除表观遗传沉默"

4. 分析RNA二级结构 → "发现IRES结构，可能影响翻译"

5. 检查miRNA → "miR-XXX可能抑制翻译"

6. 查询文献 → "类似现象在XX疾病中报道过"

结论："可能是转录后调控，建议检查..."
